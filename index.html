Semantic Profiling of the GTEx v8
SampleAttributes and SubjectPhenotypes Tables

Source: GTEx Analysis v8 public annotations — GTEx_Analysis_v8_Annotations_SampleAttributesDS.txt and GTEx_Analysis_v8_Annotations_SubjectPhenotypesDS.txt.

An LLM-agent pipeline resolved a plain-language meaning and semantic type for all 67 columns across the two tables, then evaluated the data for structural and value-level quality issues. This report summarises the inferred semantic layer and the resulting findings. The principal observation is the pervasive use of the empty string as a missing-value sentinel, which coerces numeric columns to text and affects most quantitative fields.

1Dataset and method

Two donor- and biospecimen-level annotation tables from the GTEx v8 release, linked by subject identifier. Column meanings were not provided; they were inferred from column names, value distributions, and web-grounded domain priors, then cross-checked across columns.

Domain
biobanking_genomics_repository
Cohort
980 unique human donors contributing 22,951 biospecimen samples (median ~23 samples per donor), characteristic of the GTEx project donor-tissue biobank.
Linking
Subject-level (SUBJID) to Biospecimen-level (SAMPID) registry. Note: while conceptually linked by donor, no explicit FK join was verified in this component.
Scope
Measures the provenance, quality control, and technical processing metrics of human tissue and blood biospecimens, alongside static donor demographic metadata. It does NOT measure longitudinal patient health, clinical encounter history, or molecular assay results (e.g., actual gene expression counts or variant calls).

3Principal finding — empty-string missing-value encoding

58 of 63 SampleAttributes columns contain empty strings. 8 columns are empty in every row; 50 are partially empty. Because missing entries are stored as '' rather than null, pandas types the affected numeric columns as object, so numeric comparisons (e.g. SMRIN > 7) silently fail unless the empty strings are first converted to NaN.

root cause

Structurally absent quality-control metrics

Certain tissue types (most notably Whole Blood) routinely undergo DNA isolation rather than RNA-seq. Consequently, these samples lack RNA-seq technical QC metrics (such as SMMAPRT, SMTRSCPT, etc.). These structurally missing fields are encoded as empty strings ('') instead of proper NaNs, causing the numerical columns to load as 'object' dtypes, which breaks downstream numerical queries and masking logic.

5Value-level findings

Individual values tested against each column's learned constraints (bounds, vocabularies, cross-column invariants), excluding the empty-string cases consolidated above.

violation unfiltered_metadata

SUBJID

K-562 is a well-known immortalized leukemia cell line universally used as a biological control in genomics assays, not a true subject in the GTEx post-mortem cohort (whose IDs are formatted like GTEX-XXXX). Its inclusion in the subject metadata table is a classic example of unfiltered control metadata leaking into the primary cohort data.

Remediation: Remove the K-562 row from the primary subject table or flag it explicitly as a control.

violation cross_column_inconsistency

SMMPPD

Total mapped reads (SMMPPD) must theoretically equal the sum of paired and unpaired mapped reads. In the data, SMMPPD is exactly equal to SMMPPDUN (unpaired reads) for all non-empty rows, while SMMPPDPR is non-zero. This represents a fundamental extraction or pipeline reporting error where total reads were erroneously populated with the unpaired read count.

violation cross_column_inconsistency

SMUBRID

A detailed tissue description (SMTSD) should logically map strictly 1:1 to an ontology ID (SMUBRID). However, SMUBRID '0002037' maps to two distinct tissue sites ('Brain - Cerebellum' and 'Brain - Cerebellar Hemisphere'). Since the underlying ontology contains distinct terms for both, this breaks the expected functional dependency observed in all other 53 tissue types.

violation data_artifact

SMESTLBS

SMESTLBS only contains '0' (for ~18k RNA-seq samples) and empty strings (for other assays). Despite the documentation defining it as a binary indicator flag (1=processed, 0=not), it lacks any variation to be informative, effectively functioning as a dummy/zero-inflated column. This confirms a pipeline population failure.

outlier vocab_violation

SMGEBTCHT

There is exactly one sample each for 'PCR+ 30x Coverage WGS v1 (HiSeq2000)' and 'PCR+ 30x Coverage WGS v1 (HiSeqX)'. Given the presence of dozens to hundreds of samples with identically or similarly named platforms (e.g., 79 for 'PCR+ 30x Coverage WGS (HiSeq2000)' and 579 for 'PCR+ 30x Coverage WGS v2 (HiSeqX)'), these singletons are extremely likely to be typographic variations or data entry artifacts rather than genuine distinct platforms.

outlier temporal_integrity

SMNABTCH

A nucleic acid extraction batch (SMNABTCH) groups samples processed together, meaning they should all have the same extraction date (SMNABTCHD). However, batches like BP-18079 and BP-18340 have samples recorded on conflicting dates separated by over a month (August vs September), indicating erroneous tracking or identifier reuse.

rare unfiltered_metadata

SMCGLGTH

The entire dataset (22,951 rows) consists of empty strings for this column, confirming the hypothesis that it is an unpopulated placeholder field.

Remediation: Drop the column if completely unpopulated.

plausible unfiltered_metadata

SM350NRM

The column is populated entirely by empty strings (''), confirming it functions as a placeholder or reserved field without actual data content.

Remediation: Drop column if unpopulated across all samples
