What the data means

No human wrote a data dictionary. The pipeline inferred the domain, the grain, and a plain-English meaning for each column from names, values, and web-grounded domain priors.

Domain biobanking_genomics_repository
Cohort 980 unique human donors contributing 22,951 biospecimen samples (median ~23 samples per donor), characteristic of the GTEx project donor-tissue biobank.
Linking Subject-level (SUBJID) to Biospecimen-level (SAMPID) registry. Note: while conceptually linked by donor, no explicit FK join was verified in this component.
Scope Measures the provenance, quality control, and technical processing metrics of human tissue and blood biospecimens, alongside static donor demographic metadata. It does NOT measure longitudinal patient health, clinical encounter history, or molecular assay results (e.g., actual gene expression counts or variant calls).

The bug that breaks every numeric query

One finding rose to a hard FAIL — the pipeline's strongest verdict, raised only when reverting it would recreate a real conflict in the data.

FAIL missing_key_coverage

Structurally-missing QC metrics — SMTS

Certain tissue types (most notably Whole Blood) routinely undergo DNA isolation rather than RNA-seq. Consequently, these samples lack RNA-seq technical QC metrics (such as SMMAPRT, SMTRSCPT, etc.). These structurally missing fields are encoded as empty strings ('') instead of proper NaNs, causing the numerical columns to load as 'object' dtypes, which breaks downstream numerical queries and masking logic.
Fix → Replace empty strings ('') with np.nan and strictly cast technical QC metrics to float. Ensure analysis over these columns either explicitly filters for RNA-seq isolated samples using `SMNABTCHT` or employs `.dropna()` after casting to handle the structural sparsity.

Why it matters: a quarter of the sample rows are DNA-only (e.g. Whole Blood). Their RNA-seq QC columns are blank strings, not nulls — so pandas types the whole column as object, and df.SMRIN > 7 silently misbehaves.
